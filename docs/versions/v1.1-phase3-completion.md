## Delivered Scope

### System Setup / Currency Setup Hardening

- System Setup shows company base currency and active currency status.
- Currencies, Exchange Rates, and FX Variance Policies have business UI pages.
- System Setup links to business setup pages instead of relying on Django Admin for normal finance setup.
- Local runtime `media/` uploads are ignored so upload artifacts should not enter Git going forward.

### Department General Budget Handling

- Department General Budget Setup maps department and fiscal year to a Department General Budget project.
- System Setup warns when active departments are missing current-year general budget setup.
- PR/TR validation prevents selecting a Department General Budget project owned by a different request department.
- One active department general project per department and fiscal year is enforced.

### Approval Rule Snapshot / Versioning

- Approval tasks snapshot approval rule code, rule name, rule version, step type, assigned role, and candidate pool at task creation time.
- PR/TR detail approval workflow shows the rule version context used for submitted approvals.
- Historical task context remains understandable after approval rules are edited later.

### Budget Adjustment Approval

- Manual project budget changes use Budget Adjustment Requests.
- Submitted requests do not affect budget until Finance/Admin approval.
- Approved requests post an `ADJUST` budget ledger entry.
- Rejected requests do not affect budget.

### Duplicate Actual Expense / Invoice Review

- Actual expense duplicate checks cover vendor/merchant, date, amount, and reference.
- Linked receipt/invoice file hashes are compared for duplicate evidence reuse.
- Duplicate review items do not delete or block legitimate actual expenses automatically.
- Accounting Review Detail shows runtime-computed duplicate candidates with links to PR/TR detail pages when candidates can be resolved.

Important limitation:

- Duplicate candidates are computed from current records at view time. They are not persisted as immutable duplicate snapshots.

### Attachment Retention / Deletion Control

- Request attachments are soft-deleted instead of physically removed through normal UI actions.
- Linked receipt/invoice evidence cannot be removed by normal requesters after posting.
- Accounting/Admin users can void posted or linked evidence only with a reason.
- PR/TR detail pages include Attachment History for soft-deleted or voided attachments.
- Closed request attachments remain retained for audit.

Important limitation:

- This is a void-with-reason workflow. A full attachment replacement workflow is not implemented yet.

### Optional Finance Report Drill-Down

- Project Budget Summary rows link to project budget ledgers.
- Over-Budget Exceptions and Accounting Review Aging rows link to Accounting Review Detail.
- Open Reserve rows link to the source PR/TR.
- Unmatched Card rows link to Card Transaction Detail.

Important limitation:

- This is link-based drill-down, not an advanced report framework or BI dashboard.

## Business / Accounting Validation Checklist

### Department General Budget Setup

- Finance/Admin can create one active general budget project setup per department/fiscal year.
- Duplicate active setup for the same department/fiscal year is blocked.
- System Setup shows missing current-year department setup.
- PR/TR cannot use another department's Department General Budget project.

### Approval Rule Snapshot

- Submit a PR/TR under approval rule version A.
- Edit the approval rule to version B.
- Existing request still shows version A snapshot context.
- New request uses the newer rule context.

### Budget Adjustment Request

- Submitted adjustment does not change budget.
- Approved adjustment creates an `ADJUST` ledger entry.
- Rejected adjustment does not affect budget.
- Unauthorized users cannot approve/post adjustment.

### Duplicate Actual Expense / Invoice Review

- Same vendor/date/amount/reference creates duplicate review.
- Same receipt/invoice file hash creates duplicate review.
- Review detail shows candidate information and links to source PR/TR detail.
- Accounting can resolve or approve exception.
- Actual expense records are not deleted automatically.

### Attachment Retention / Void

- Draft attachment can be deleted by authorized user.
- Linked receipt/invoice evidence cannot be removed by normal requester after posting.
- Accounting/Admin void requires reason.
- Voided attachment remains visible in Attachment History.
- Closed request attachment deletion is blocked.

### Finance Report Drill-Down

- Project row opens project budget ledger.
- Open Reserve row opens source PR/TR.
- Over-Budget and Review Aging rows open Accounting Review Detail.
- Unmatched Card row opens Card Transaction Detail.

### Permission Validation

- Normal requesters cannot access finance setup pages.
- Finance/Admin can access System Setup and finance setup pages.
- Accounting can review duplicate and attachment evidence issues.
- Navigation visibility remains consistent with direct URL permission checks.

## Remaining Accepted Risks

- Duplicate candidate links are runtime-computed, not immutable persisted snapshots.
- There is no dedicated invoice number field yet; reference number is still used as the invoice/reference proxy.
- Duplicate detection warns and routes to accounting review; it does not implement duplicate policy `BLOCK`.
- Department General Budget does not yet include a PR/TR "general spend" flag or auto-default behavior.
- Attachment Retention supports void-with-reason and history, but not full replacement workflow.
- Finance Report drill-down is link-based; Excel export and advanced reporting remain future work.
- Broader role-matrix hardening for Accounting Clerk, Accounting Manager, Finance Admin, and System Admin remains a production readiness item.
