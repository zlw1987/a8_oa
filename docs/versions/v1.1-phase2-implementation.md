# V1.1 Phase 2 Implementation Status

V1.1 Phase 2 implements the selected Expense Control And Approval Operations scope.

## Implemented Scope

### Line-Level Receipt Matching UI

- PR detail and TR detail actual expense sections show receipt and invoice status per actual expense line.
- Users with attachment/accounting permission can upload a receipt/invoice directly to an actual expense line.
- Users can link an existing request-level attachment to a specific actual expense line.
- Missing receipt review detail now shows the exact actual expense line and evidence status.
- Linking required line-level support can resolve the missing receipt review path.

### Direct Project Cost Policy Workflow

- Added Direct Project Cost Policy setup pages.
- Added Setup navigation and System Setup shortcuts for Direct Project Cost Policies.
- Company card allocation to Project Direct Cost resolves the policy before posting.
- Policy actions:
  - `ALLOW`
  - `REVIEW`
  - `REQUIRE_PROJECT_OWNER_APPROVAL`
  - `BLOCK`
- `REVIEW` and `REQUIRE_PROJECT_OWNER_APPROVAL` create accounting review items.
- `BLOCK` prevents direct project cost allocation.
- Card Transaction Detail shows direct project cost policy result and project owner review status.

### Approval Delegation UI

- Added My Delegations.
- Added Create/Edit Delegation.
- Added Active Delegations admin/finance view.
- Delegated tasks appear in My Tasks during the active date range.
- Delegated approval history records that the delegate acted on behalf of the original approver.
- Requester self-approval remains blocked.

### Approval Reassignment And Overdue Handling

- Finance/Admin users can reassign open approval tasks with a required reason.
- My Tasks continues to highlight overdue assigned and pool tasks.
- Added `process_approval_escalations` command as an operational alias for the existing escalation process.

## Data / Migration Notes

- `finance.AccountingReviewItem.policy_action` was widened to support direct project cost action names such as `REQUIRE_PROJECT_OWNER_APPROVAL`.
- Migration added: `finance/migrations/0005_alter_accountingreviewitem_policy_action.py`.
