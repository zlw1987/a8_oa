# V0.6 Multi-Currency Foundation

## Purpose

V0.6 introduces the foundation for currency-safe finance control. The goal is to prevent the system from silently mixing USD, TWD, EUR, or other currencies in approval, actual expense, budget ledger, card reconciliation, and finance reporting.

## Core Decision

Company base currency is USD.

Base currency is used for:

- Project budget control.
- Budget reserve, consume, and release ledger entries.
- Over-budget comparison.
- Finance Reports.
- Management reporting.

Transaction currency is preserved for audit and reimbursement detail.

## Implemented In This Version

- Added reusable currency helpers in `common/currency.py`.
- Added Currency and ExchangeRate models.
- Added FXVariancePolicy setup model.
- Added transaction/base currency snapshot fields to PR/TR actual expense records.
- Added transaction/base currency snapshot fields to Accounting Review Items.
- Added source transaction currency fields to ProjectBudgetEntry.
- Added company card transaction/base currency fields.
- Company card posted/base amount is treated as the allocation and budget-control amount.
- Actual expense policy evaluation now distinguishes:
  - FX_VARIANCE
  - SPENDING_OVERRUN
  - BASE_CURRENCY_VARIANCE
- FX variance can route through FXVariancePolicy instead of normal over-budget policy.
- Finance Reports now show USD base-currency totals.
- Finance Reports preserve and display original transaction currency where relevant.
- User guides were updated in English and Chinese.

## Current Behavior

For normal USD expenses:

- Transaction amount and base amount are both USD.
- Exchange rate is 1.00000000.
- Finance Reports display USD totals.

For foreign-currency reimbursement:

- The source record stores transaction currency and transaction amount.
- The source record stores USD base amount and exchange-rate snapshot.
- If no base amount is provided, the system looks up an ExchangeRate record.
- If no exchange rate is available, posting is blocked with a clear validation error.

For company card foreign transactions:

- The card posted USD amount is treated as the authoritative base amount.
- Original foreign currency and original amount are preserved when available.
- Budget consumption and allocation use the USD base amount.

## Variance Classification

If approved and actual transaction currencies match:

- Actual transaction amount unchanged but USD base amount increased: FX_VARIANCE.
- Actual transaction amount increased: SPENDING_OVERRUN.

If approved and actual transaction currencies do not match:

- The system compares base currency amounts and classifies as BASE_CURRENCY_VARIANCE.
