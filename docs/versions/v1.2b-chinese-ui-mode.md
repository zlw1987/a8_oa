# V1.2B Chinese UI Mode Foundation

## Status

V1.2B is implemented as the first controlled localization foundation.

This release enables Django i18n infrastructure and adds an English / Chinese language switcher. English remains the default language.

## Delivered Scope

- `USE_I18N` remains enabled.
- `LANGUAGES` now includes English and Simplified Chinese.
- `LOCALE_PATHS` points to the project `locale/` directory.
- `LocaleMiddleware` is enabled after `SessionMiddleware` and before `CommonMiddleware`.
- `/i18n/setlang/` is wired through Django's built-in language switching URL.
- The base HTML `lang` attribute follows the active language.
- The top navigation includes an English / 中文 switcher that posts with CSRF protection and preserves the current page through `next`.
- The first translation batch covers top navigation, Dashboard shell/cards, Finance Reports, Department Spending drill-down, Accounting Review Queue/Detail key labels, System Setup key labels, and high-traffic Python-generated labels in navigation/dashboard/accounting review presentation/filter forms.

## Locale Files

The following files are intentionally included:

- `locale/zh_Hans/LC_MESSAGES/django.po`
- `locale/zh_Hans/LC_MESSAGES/django.mo`

The `.mo` file is committed intentionally because local development, Codex, and the GoDaddy testing environment may not have GNU gettext / `msgfmt` available.

When translations change, developers should install GNU gettext and run:

```powershell
python manage.py makemessages -l zh_Hans
python manage.py compilemessages
```

In the current environment, Django's gettext commands failed because GNU gettext tools were missing:

```text
CommandError: Can't find msguniq. Make sure you have GNU gettext tools 0.19 or newer installed.
CommandError: Can't find msgfmt. Make sure you have GNU gettext tools 0.19 or newer installed.
```

The `.mo` file was generated from the `.po` file so Chinese UI rendering works in the app and targeted tests.

## Limitations

- This is not full product-wide localization yet.
- Full PR/TR detail pages are not fully translated.
- Most setup forms are not fully translated.
- Many validation messages and workflow messages in Python remain English.
- Django Admin remains English/default Django behavior.
- Database-stored enum/status values, codes, and user-entered data are not translated or changed.
- V1.1 Phase 1, Phase 2, and Phase 3 workflows were not changed.
