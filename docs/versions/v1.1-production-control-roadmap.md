# V1.1 Production Control Roadmap

## Release Structure

V1.1 is organized as a controlled roadmap, not a single broad sprint.

### V1.1A: Admin And Permission Foundation

Implemented foundation:

- Real System Setup landing page.
- Admin dropdown no longer uses System Setup as a duplicate Django Admin alias.
- Role / permission matrix documented in `common.permissions`.
- Navigation and finance views begin using shared permission functions.
- Setup form help text added for Approval Rules, Approval Steps, Over-Budget Policies, and Receipt Policies.
- Approval Rule Step Editor changed from a wide editable table to step cards / accordion sections.

### V1.1B: Financial Integrity

Implemented foundation:

- AccountingPeriod model with OPEN / CLOSING / CLOSED.
- Closed-period guard service for actual expense posting and card allocation.
- PR/TR reopen-for-correction fields and model methods.
- PR/TR refund / credit entry support as separate negative actual records.
- Refund / credit budget impact uses negative CONSUME ledger entries.

Remaining hardening:

- Dedicated period close UI.
- Post-close adjustment UI and approval flow.
- Rich correction history screen.
- Refund forms in PR/TR detail pages.

### V1.1C: Expense Control And Approval Operations

Implemented foundation:

- ActualExpenseAttachment model for line-level receipt/invoice linkage.
- CardAllocationAttachment model.
- Receipt policy evaluation now checks line-level actual expense attachments.
- DirectProjectCostPolicy model.
- Direct project card allocation applies policy and creates review items when required.
- ApprovalDelegation and ApprovalEscalationPolicy models.
- Approval task claim/activation/reassign logic understands delegation and blocks requester self-approval.

Remaining hardening:

- Line-level attachment UI.
- Direct project cost policy list/form UI.
- Delegation self-service UI.
- Escalation execution command.

### V1.1D: Reporting And Setup UX

Implemented foundation:

- Finance Reports CSV export.
- Finance Reports export includes base currency and original transaction currency columns where available.
- Report rows already link to project, request, accounting review, and card source pages where available.

Remaining hardening:

- Excel export.
- Saved filters.
- Column sorting.
- Page size selectors across all list pages.
- Sticky headers where useful.

## Do Not Include Yet

- AI receipt OCR.
- Full mobile app.
- Vendor onboarding.
- PO / invoice / 3-way match.
- ACH reimbursement payment.
- GL / ERP integration.
- Advanced BI dashboard.
